# DesignTest: "НЕТРАННЕР - Эхо Информации"

Интерактивный браузерный проект по домашнему заданию курса "Теория информации и протоколы безопасного обмена данными". Проект оформлен как киберпанк-игра из трех операций, каждая из которых закрывает один пункт БДЗ:

1. Творческое объяснение понятия "количество информации".
2. Средство расчета полной взаимной информации по матрице совместных вероятностей.
3. Реализация протокола шифрования и дешифрования 3DES.

## Версия 2: обучающая киберпанк-головоломка

Во второй итерации проект оформлен как полноценный netrun: появились игровые последствия, checkpoint/rollback, выбор сложности, Story Panels с монологом героя и финал `Obsidian Breach`, открывающийся после puzzle-этапов.

Главная идея осталась прежней: проект закрывает три задания БДЗ. Новая игровая оболочка делает прохождение глубже: правильные решения дают `Access Fragments`, ошибки поднимают `ICE Heat` и `Trace`, критические промахи режут `Integrity`, а поражение переводит игрока на экран `ICE TRACE COMPLETE` с возвратом к последнему checkpoint.

### Глобальное состояние забега

Состояние хранится в `localStorage` и включает:

- `ICE Heat` - насколько сильно разогрет защитный контур.
- `Trace` - близость BurnICE к физическому каналу.
- `Integrity` - запас нейрошунта.
- `Access Fragments` - ключевые фрагменты `BIT`, `ENTROPY`, `MUTUAL`, `CIPHER`.
- `Checkpoint` - точка rollback после крупной операции.
- `failCount` - количество откатов.
- `terminalLog` - журнал событий забега.

В интерфейсе добавлены кнопки `Reset Run` и `Restore Checkpoint`. При `heat >= 100`, `trace >= 100` или `integrity <= 0` показывается defeat screen:

```text
ICE TRACE COMPLETE
Соединение сгорело.
Rollback to last checkpoint?
```

### Сложность

- `Normal`: больше подсказок, мягче штрафы, таймеры длиннее.
- `Hardline`: меньше подсказок, выше штрафы, таймеры короче; финальная ошибка откатывает глубже.

### Story Panels

Добавлены сюжетные плашки героя `Johnny`: портрет, имя, короткий монолог и статус `Calm / Nervous / Overheated / Focused / Breach Mode`. Плашки появляются после boot, перед миссиями, при ошибках, перегреве, rollback, поражении и успехе.

Чтобы вставить портрет героя, положите файл:

```text
assets/hero-portrait.png
```

Если файла нет, интерфейс показывает встроенную неоновую заглушку и продолжает работать.

### Новые puzzle-механики

- `Signal Value Puzzle`: выбор лучшего сигнала по вероятности, количеству информации, стоимости перехвата и риску ICE. Успех даёт `BIT`.
- `Entropy Lock`: три настройки распределения вероятностей с live `H(X)`, `Hmax`, `H/Hmax` и таймером. Успех даёт `ENTROPY`.
- `Matrix Breach`: три gated-матрицы `P(X,Y)` поверх старого калькулятора взаимной информации. Успех даёт `MUTUAL`.
- `Cipher Tunnel Assembly`: выбор параметров 3DES-туннеля: `CBC`, `random IV`, `EDE`, `PKCS#7`, сильный 24-байтовый ключ. Успех даёт `CIPHER`.
- `Obsidian Breach`: финальная последовательность, открывается только после всех четырёх fragments и объединяет темы количества информации, энтропии, взаимной информации и 3DES.

### Сценарий демо

1. Boot sequence.
2. Выбор сложности `Normal` или `Hardline`.
3. `Signal Value Puzzle` - получить `BIT`.
4. `Entropy Lock` - получить `ENTROPY`.
5. `Matrix Breach` - получить `MUTUAL`.
6. `Cipher Tunnel Assembly` - получить `CIPHER`.
7. Показать поражение: несколько раз неверно проверить `Entropy Lock`, дождаться `ICE TRACE COMPLETE`, нажать `Restore from checkpoint`.
8. Финальный `Obsidian Breach`.
9. Расширенный сертификат `BREACH COMPLETE`.

## Запуск

Проект не требует сборки. Достаточно отдать папку через любой статический HTTP-сервер:

```bash
cd DesignTest
python3 -m http.server 8100 --bind 127.0.0.1
```

После запуска открыть:

```text
http://127.0.0.1:8100/Netrunner.html
```

Открытие через `file://` нежелательно, потому что JSX-файлы подключаются как отдельные скрипты через Babel Standalone.

## Общая игровая оболочка

Проект построен как мини-игра с тремя "операциями". Игрок проходит уровни, отмечает их выполненными и получает финальный экран-сертификат.

Реализованные общие механики:

- boot sequence: псевдотерминальная загрузка с поэтапным выводом строк;
- главное меню с тремя карточками операций;
- боковая навигация между меню, уровнями и финалом;
- верхний статус с текущей операцией, временем и уровнем угрозы ICE;
- нижняя статусная строка с текущим состоянием операции;
- отметка прохождения каждого уровня;
- финальный экран, который открывает полный сертификат только после прохождения всех трех операций;
- печать сертификата через `window.print()`;
- единая визуальная тема: неоновые панели, сетка, шум, сканлайны, "дождь" данных.

## Уровень 1: "Колыбель Шеннона"

Цель уровня: выполнить творческое задание N1 и показать понимание количества информации как меры снятия неопределенности.

### Механика I: сортировка сигналов

Игрок получает несколько сценариев и выбирает сигнал с наибольшей средней информативностью.

Реализовано:

- три раунда с разными сюжетными ситуациями;
- четыре варианта ответа в каждом раунде;
- расчет энтропии для каждого варианта через `H(X) = -sum p(x) log2 p(x)`;
- подсветка правильного и неправильного выбора;
- объяснение, почему выбранный вариант дает больше или меньше информации;
- счетчик результата;
- автоматический переход к следующей части уровня после завершения раундов.

Смысл механики: игрок видит, что самый информативный сигнал не самый длинный и не самый "громкий", а тот, который сильнее всего уменьшает неопределенность.

### Механика II: лаборатория энтропии

Интерактивная лаборатория для распределений с 2, 3, 4, 6 и 8 исходами.

Реализовано:

- выбор числа исходов;
- ползунки вероятностей;
- автоматическое перераспределение остальных вероятностей при изменении одного исхода;
- кнопка равномерного распределения;
- кнопка перекошенного распределения;
- расчет `H(X)`;
- расчет максимальной энтропии `Hmax = log2(n)`;
- расчет эффективности `H/Hmax`;
- визуализация "облака вероятностей";
- демонстрация, что равномерное распределение дает максимум информации, а предсказуемое распределение снижает энтропию.

### Механика III: отсек сжатия

Демонстрация связи энтропии и сжимаемости.

Реализовано:

- несколько текстовых примеров: повторяющаяся реклама, сенсорный лог, смысловое сообщение, случайный поток;
- подсчет частот символов;
- расчет энтропии символов;
- подсчет количества уникальных символов;
- простая оценка RLE-сжатия;
- пояснение, почему повторяющиеся строки несут меньше информации на символ, чем случайный поток.

Смысл механики: показать, что избыточность снижает информационную плотность, а сжатие можно воспринимать как практическую работу с энтропией.

### Механика IV: творческое эссе

Встроенный письменный разбор в художественной форме.

В эссе раскрыто:

- информация как снятие неопределенности;
- отличие количества информации от размера сообщения;
- формула Шеннона;
- примеры с честной и смещенной монетой;
- связь сжатия, канального кодирования, криптографии, машинного обучения и Big Data с теорией информации;
- упоминание физического смысла бита через принцип Ландауэра;
- итоговая метафора: важна не громкость канала, а способность сигнала удивить.

## Уровень 2: "Решетка взаимности"

Цель уровня: реализовать средство расчета полной взаимной информации в сложной системе, состоящей из двух зависимых систем. Входные данные - матрица вероятностей сочетания состояний систем.

### Ввод матрицы

Реализовано:

- таблица совместных вероятностей `P(X, Y)`;
- редактирование каждой ячейки;
- запрет отрицательных значений при ручном вводе;
- добавление и удаление строк;
- добавление и удаление столбцов;
- кнопка нормализации матрицы к сумме 1;
- отображение текущей суммы всех ячеек;
- тепловая подсветка ячеек по величине вероятности;
- произвольный размер матрицы от 2x2 и выше.

### Пресеты

Реализованы готовые сценарии:

- независимые системы, где `I(X;Y) ~= 0`;
- идеальная связь, где `Y = X`;
- слабая связь;
- сюжетная сетка наблюдения `ShenTech 4x4`;
- шумный канал `3x3`.

### Расчеты

Все расчеты выполняются в `info-theory.js`.

Реализованные метрики:

- маргинальное распределение `p(X)`;
- маргинальное распределение `p(Y)`;
- энтропия `H(X)`;
- энтропия `H(Y)`;
- совместная энтропия `H(X,Y)`;
- полная взаимная информация `I(X;Y)`;
- условная энтропия `H(X|Y)`;
- условная энтропия `H(Y|X)`;
- нормированная взаимная информация `NMI = 2I / (H(X) + H(Y))`.

Основные формулы показаны прямо на уровне:

```text
H(X) = -sum_i p(x_i) log2 p(x_i)
H(Y) = -sum_j p(y_j) log2 p(y_j)
H(X,Y) = -sum_i sum_j p(x_i,y_j) log2 p(x_i,y_j)
I(X;Y) = sum_i sum_j p(x_i,y_j) log2(p(x_i,y_j) / (p(x_i)p(y_j)))
I(X;Y) = H(X) + H(Y) - H(X,Y)
```

### Визуализация

Реализовано:

- столбики маргинальных распределений `p(X)` и `p(Y)`;
- диаграмма Венна для бюджета энтропии;
- выделение общей части `I(X;Y)`;
- вывод остаточных неопределенностей `H(X|Y)` и `H(Y|X)`;
- текстовый вердикт по силе зависимости.

## Уровень 3: "Лом Черного Льда"

Цель уровня: разработать реализацию протокола шифрования и дешифрования 3DES.

### Криптографическое ядро

Реализация находится в `tdes.js` и не использует внешние криптографические библиотеки.

Реализовано:

- стандартные таблицы DES: `PC1`, `PC2`, `IP`, `FP`, `E`, `P`, `S-box`;
- генерация 16 подключей DES;
- функция Фейстеля;
- DES-шифрование и DES-дешифрование 64-битного блока;
- Triple DES в схеме EDE:
  - шифрование: `E_K1 -> D_K2 -> E_K3`;
  - дешифрование: `D_K3 -> E_K2 -> D_K1`;
- 24-байтовый ключ, разделенный на `K1`, `K2`, `K3`;
- поддержка ключа как 48 hex-символов или произвольной строки;
- PKCS#7 padding для 8-байтовых блоков;
- режимы ECB и CBC;
- IV для CBC;
- конвертация строк, байтов и hex.

### Вкладка I: шифроядро

Реализовано:

- ввод режима `ECB` или `CBC`;
- ввод IV для CBC;
- ввод ключа;
- отображение `K1`, `K2`, `K3` в hex;
- ввод открытого текста;
- шифрование текста в hex;
- дешифрование hex обратно в текст;
- демонстрация ошибки при попытке расшифровать неверным ключом;
- терминальный лог операций;
- отображение результата шифрования и дешифрования.

### Визуализация EDE

Реализовано:

- пошаговая анимация первого блока;
- отдельные карточки стадий `E_K1`, `D_K2`, `E_K3`;
- показ входа и выхода каждого этапа в hex;
- пояснение про двухключевой 3DES, трехключевой 3DES и слабый случай `K1 = K2 = K3`.

### Вкладка II: работа с файлами

Реализовано:

- загрузка произвольного файла как сырых байтов;
- шифрование файла через тот же 3DES-пайплайн;
- скачивание результата в формате `.3des`;
- загрузка `.3des`;
- расшифровка файла тем же ключом, режимом и IV;
- скачивание восстановленного исходного файла;
- логирование размера входных и выходных данных.

### Вкладка III: демонстрация перебора

Реализовано:

- имитация перебора случайных 24-байтовых ключей;
- счетчик попыток;
- показ текущего кандидата ключа;
- оценка прогресса относительно `2^168`;
- пояснение, что meet-in-the-middle снижает фактическую стойкость примерно до `2^112`;
- индикатор "нагрева" и автоматическая остановка демонстрации.

Эта вкладка не является реальным атакующим инструментом. Она нужна для объяснения масштаба пространства ключей.
